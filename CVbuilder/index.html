How to create your CV from scratch

Step 1 — Start with a blank template

Go to File → New CV… and click Start blank CV. This loads a minimal JSON template with the most common CV sections already defined (basics, education, publications, skills, languages, etc.).

Step 2 — Fill in your basics

Click the basics section in the left sidebar. Fill in your first name, last name, email, homepage, location, and optionally a photo path. These populate the \firstname, \familyname, \email, and \homepage commands in moderncv.

Step 3 — Add entries to each section

Click any section (e.g., education) in the sidebar. Use the Add entry button to create a new item. Fill in the fields — each field maps to a column in a \cventry command or equivalent LaTeX macro.

  • Toggle Included on each entry to include or exclude it from the LaTeX output.
  • Use Duplicate to copy a filled-in entry as a starting point for a similar one.
  • Use Delete to remove an entry permanently.

Step 4 — Rename labels and section titles

Each field in the editor has a label on the left (e.g., degree). You can rename it — this controls how it appears in the LaTeX output. You can also set a custom Output title for each section (e.g., rename education to Academic Training).

Step 5 — Preview the LaTeX

Click the LaTeX tab at the top of the workspace to see the generated .tex source in real time. Use the JSON tab to inspect the raw data structure.

Step 6 — Save your progress

Use File → Export data (JSON) regularly to save your work as a .json file. The banner at the top of the sidebar turns red whenever there are unsaved changes — it turns green once you export. Next time, reload this file with File → Upload data…

Step 7 — Download the LaTeX file

When your CV is ready, go to File → Download .tex to get the compiled LaTeX file. Open it in Overleaf or compile locally with TeX Live / MiKTeX. The document requires the moderncv package.

How to use CV Builder

1. Upload your CV data

Go to File → Upload data… and select a .json file that follows the CV schema (see below). The app will load all sections automatically.

If you don't have a JSON file yet, create one manually or export a blank template from this tool and fill it in.

2. Navigate sections

The left sidebar lists every section found in your JSON (e.g., basics, education, publications). Click a section to open its editor in the main workspace.

3. Edit entries

  • Use the form fields to edit any value directly.
  • Toggle the Include checkbox on each entry to include or exclude it from the generated LaTeX.
  • Use Duplicate to copy an entry, or Delete to remove it.
  • Rename property labels in the left column — these are the labels used in the LaTeX output.

4. Include/exclude sections

Open a section and uncheck the Include toggle at the top to hide the entire section from the LaTeX output. You can also rename the output section title.

5. Preview and export

  • LaTeX tab — live preview of the generated .tex file.
  • JSON tab — view the current data as raw JSON.
  • File → Download .tex — download the generated LaTeX file.
  • File → Export data (JSON) — save your edited data as a .json file. Do this regularly to avoid losing work!

6. Unsaved changes indicator

The banner at the top of the sidebar turns red whenever your data has been modified since the last JSON export. It turns green once you export. This is a reminder — the app does not auto-save.

CV JSON schema (basic structure)

Your JSON file should be an object with named section keys. Common keys and their types:

  • basics — object with fields like firstname, lastname, email, location, homepage, photo, title.
  • education — array of objects with degree, institution, start, end, description, dissertation.
  • publications — array of objects with authors, title, venue, year.
  • abstracts — same as publications plus type.
  • skills — object where each key is a group name and the value is an array of skill strings or {name, selected} objects.
  • languages — array of objects with language and proficiency.
How to use Templates

What is a template?

A template in CV Builder controls how the LaTeX document is structured: the title, the moderncv style, optional extra preamble commands, optional footer lines, and the mappers that define how each section is rendered.

Edit template (Template → Edit template)

  • CV title — text shown in the \title{} command.
  • moderncv style — e.g. classic, casual, banking, oldstyle, fancy.
  • Extra preamble lines — any LaTeX commands you want inserted after \moderncvstyle, one per line (e.g. \moderncvcolor{blue}).
  • Extra footer lines — LaTeX commands inserted just before \end{document}.

Click Apply to update the LaTeX preview immediately.

Edit mappers (Template → Edit mappers)

Mappers tell the app which LaTeX rendering strategy to use for each section. The editor shows a JSON object where keys are section names and values are strategy identifiers:

  • cventry_education — \cventry with date (parentheses), degree, institution, dissertation.
  • cventry_research — \cventry with date (dash), organization, role, description.
  • cventry_work — \cventry with date (dash), role, department, organization.
  • cventry_teaching — \cventry with course area, institution, level, courses.
  • publications / abstracts — hanging-indent bibliography style.
  • awards — \cvitemwithcomment with category, description, year.
  • skills — \cvitemwithcomment per skill group.
  • languages — \cvitemwithcomment per language with proficiency.
  • continuing_education — \cvitem with type and details.

Click Apply to regenerate the LaTeX with the updated mappings.

Import template (Template → Import template)

Load a previously exported cv_template.json file. This restores the CV title, style, preamble, footer, and all mappers at once — useful for switching between CV variants.

Export template (Template → Export template)

Save the current template configuration as cv_template.json. Share it with colleagues or use it to quickly restore your settings on a different machine or browser session.
